# 网感训练板

> **把网感从玄学变成可训练的能力。** 通过「拆解 → 预测 → 验证」的闭环训练系统，让内容创作者对爆款的判断力从"凭感觉猜"升级为"可校准的预测"。

[👉 在线使用](https://lillltachen.github.io/net-sense-trainer/)

---

## 核心理念

网感不是天赋——它可以被拆解为**钩子强度、情绪触发、结构完整度**三个可量化维度。绝大多数创作者靠直觉判断一条内容会不会爆，但直觉不可复现、不可校准、不可迁移。这个系统把判断过程显性化，让每次判断都能被记录、被对比、被改进。

### 5D 网感训练法

1. **Detect（识别）** — 发现值得分析的内容，一键采集入库
2. **Deconstruct（拆解）** — AI 辅助从钩子、情绪、结构三个维度拆到底
3. **Decide（判断）** — 在"不知道答案"的状态下做判断
4. **Diff（对比）** — 提交后对比参考答案，记录准确度
5. **Drill（强化）** — 薄弱平台多练，雷达图追踪成长

## 功能模块

### 📝 内容采集
- 粘贴链接 AI 自动解析（自动识别 22 个中国主流平台）
- 浏览器 Bookmarklet 一键采集
- 手动录入 / JSON 导入导出

### 🤖 AI 评分引擎（六步推理链）
不是简单"打分 1-10"，而是模拟资深内容运营的判断路径：
1. 识别赛道（职场/美食/情感/科技…）
2. 赛道基准评分（5 分 = 该赛道平均，非绝对质量分）
3. 自适应权重分配（短视频钩子 50%，知乎深度文结构 60%）
4. 百分位排名（在同赛道能打败多少竞品）
5. 传播潜力评估（独立于质量分的分享/评论驱动力）
6. 评分理由（强制含六要素，不准写空洞评语）

### 🔍 拆解分析
- 钩子分析：是什么让用户停下来的？
- 情绪触发：引发了什么情绪？
- 可复用结构：内容是怎么组织的？
- 评论区洞察：高赞评论在说什么？
- 传播驱动力 TOP3 识别

### 🎮 训练中心（三种模式）
| 模式 | 训练目标 | 玩法 |
|------|---------|------|
| **A：拆解填空** | 分析深度 | 看标题 → 自己拆钩子/情绪/结构 → 对比答案 |
| **B：维度聚焦** | 判断直觉 | 判断内容的最强驱动力维度 |
| **C：评论预言** | 用户共情 | 预测最高赞评论方向 |

### 📊 统计面板
- 各平台网感雷达图（六边形可视化强弱项）
- 三种训练模式准确率追踪
- 今日/累计训练数 + 准确率趋势
- 最近训练记录

## 平台差异化评分

22 个中国内容平台使用不同权重——不存在万能标题：

| 平台 | 钩子权重 | 情绪权重 | 结构权重 | 特征 |
|------|---------|---------|---------|------|
| 微博 | 45% | 35% | 20% | 前 5 字定生死 |
| 抖音 | 50% | 30% | 20% | 前 3 秒完播率决定一切 |
| 小红书 | 30% | 40% | 30% | 情绪共鸣是裂变核心 |
| 知乎 | 15% | 25% | 60% | 结构性信息密度是护城河 |
| 公众号 | 20% | 30% | 50% | 标题打开率 + 完读率双指标 |
| B 站 | 35% | 30% | 35% | 标题封面双钩 + 内容深度 |

> 一个在微博爆了的标题原样复制到知乎大概率没人看——平台间差异不是"风格"问题，是内容消费场景的根本不同。

## 技术架构

- **前端**：单 HTML 文件（~3670 行），零构建，双击即用
- **存储**：localStorage 纯本地，数据完全私有，API Key 仅存浏览器
- **AI**：OpenAI 兼容 API，支持多配置切换（OpenAI / Kimi / 自定义）
- **后端（可选）**：FastAPI + SQLite，含热点爬虫
